B. 最小改动：Huber Loss
=====================
1. 在 `main_model.py` 中新增一个函数 `huber_loss(residual, delta)`，按照论文定义实现 Huber 损失：
   ```
   def huber_loss(residual, delta=1.0):
       abs_res = torch.abs(residual)
       quadratic = torch.where(abs_res <= delta, 0.5 * residual ** 2, delta * (abs_res - 0.5 * delta))
       return quadratic
   ```
2. 修改 `calc_loss` 和 `calc_loss_valid` 中对残差的处理：
   - 代替原有的 `loss = (residual ** 2).sum() / (num_eval if num_eval > 0 else 1)`。
   - 改为：`loss = huber_loss(residual).sum() / (num_eval if num_eval > 0 else 1)`。
   - 从配置文件读取一个新的超参 `huber_delta`（默认值 1.0），并将其传递给 `huber_loss`（即 `huber_loss(residual, delta=self.huber_delta)`）。若配置中未提供该值，就使用默认值。
3. 在模型的 `__init__` 方法或配置解析处，增加 `self.huber_delta = config["train"].get("huber_delta", 1.0)`。保持向下兼容：旧配置无需指定此参数即可运行。

C. 最小改动：可选短 horizon 预测头
=====================
1. 在 `main_model.py` 中，找到模型输出位于扩散模型之后的地方（如 `predicted = ...`），目前只输出一个张量形状 `(B, K, L)`。
2. 定义一个新的线性层 `self.short_head = nn.Linear(self.pred_len, short_pred_len, bias=True)`，其中 `short_pred_len` 可在配置文件中设置（建议为 8），默认情况下不启用额外输出。
3. 在 `calc_loss` 中，如果配置项 `use_short_horizon_head` 为 `True`，则：
   - 对 `predicted[:, :, -self.pred_len:]` 做 `self.short_head` 投射，得到 `(B, K, short_pred_len)` 的短 horizon 预测。
   - 从 `observed_data` 中提取对应的真实值 `(observed_data[:, :, -short_pred_len:])`，计算 Huber 损失并累加到原有损失中。
   - 可使用 `short_head_weight` 超参控制两个损失的比例，例如：
     ```
     loss_main = huber_loss(residual, delta=self.huber_delta).sum() / num_eval
     loss_short = huber_loss(predicted_short - target_short, delta=self.huber_delta).sum() / (B*K*short_pred_len)
     loss = loss_main + config["train"].get("short_head_weight", 1.0) * loss_short
     ```
4. 在配置文件(如某个 yaml)中新增：
   ```
   train:
     use_short_horizon_head: false   # 默认不启用
     short_pred_len: 8               # 短期预测长度
     short_head_weight: 1.0          # 短期损失权重
     huber_delta: 1.0                # Huber 损失的 delta
   ```
